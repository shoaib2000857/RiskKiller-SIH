# Docs Overview (docs/)

## Purpose
Centralize architecture, integration guides, and operational references.

## Documents
- architecture.md
  - High-level system architecture and data flow.
  - Core services and extensibility roadmap.

- FEDERATED_BLOCKCHAIN.md
  - Detailed federated ledger design and API usage.
  - Validation and tamper detection strategy.

- OLLAMA_SETUP.md
  - Local LLM setup steps and model pull commands.

- OLLAMA_INTEGRATION.md
  - Integration notes and expected output format.

- project_summary.md
  - High-level project summary and next steps.

## Support Artifacts
- submission_logs_schema.sql / submission_logs_roles.sql
  - SQL references for audit logging.

- insert_submission_log.py
  - Helper script for inserting sample logs.

## Design Intent
Keep operational knowledge close to the codebase so teammates can onboard and extend the system quickly.
